# Release Notes - v0.6.0

**Fecha:** 2026-03-04
**Nombre:** Deployment a Produccion + Rediseno UI Glassmorphism
**Autor:** Samael Hernandez - UNIR 2026

## Resumen

La version 0.6.0 representa el primer deployment a produccion del Etiquetador Morfosintactico. Incluye configuracion completa de Docker Swarm con Traefik como reverse proxy, integracion con Supabase para persistencia, rediseno completo de la UI con glassmorphism design system inspirado en Apple HIG, y mejoras significativas en la pagina de Probabilidades. Se realizo auditoria de seguridad confirmando zero secretos expuestos.

---

## Cambios Detallados

### 1. Deployment a Produccion con Docker Swarm + Traefik

**Archivos:**
- `docker-stack.yml` (nuevo)
- `frontend/Dockerfile`
- `frontend/nginx.conf`
- `frontend/src/environments/environment.prod.ts` (nuevo)
- `frontend/angular.json`
- `backend/requirements.txt`

#### Docker Swarm Stack

Se creo `docker-stack.yml` para deployment en produccion con Docker Swarm:

- Traefik como reverse proxy con TLS automatico
- Servicio de frontend (Angular + Nginx) con proxy a backend
- Servicio de backend (FastAPI + Uvicorn)
- Overlay network para comunicacion inter-servicio

#### Configuracion de Produccion Frontend

- `environment.prod.ts` con `apiUrl` vacio para same-origin proxy via Nginx
- `angular.json` con `fileReplacements` para sustituir environment en build de produccion
- `@source` directives en `styles.css` para que Tailwind v4 detecte clases en Docker

#### Nginx para Produccion

- Resolver DNS Docker (`127.0.0.11`) para resolucion de servicios en overlay network
- `proxy_pass` con variables para backend
- CSP headers actualizados para permitir Google Fonts (`fonts.googleapis.com`, `fonts.gstatic.com`)

#### Backend

- Dependencias `requests` y `python-multipart` agregadas a `requirements.txt`
- Integracion completa con Supabase para persistencia remota del modelo HMM

### 2. Rediseno UI Glassmorphism

**Archivos:** Todos los componentes frontend

Se rediseno toda la interfaz de usuario con un design system basado en glassmorphism, inspirado en Apple Human Interface Guidelines:

- Fondos con blur y transparencia
- Tarjetas con efecto glass
- Consistencia visual en todos los componentes
- Soporte completo para dark mode

### 3. Fixes de Deployment

| Problema | Solucion |
|----------|----------|
| `.postcssrc.json` faltante en VPS | Agregado archivo de configuracion PostCSS |
| DNS no resuelve servicios Docker | nginx resolver `127.0.0.11` para overlay network |
| Google Fonts bloqueado por CSP | CSP actualizado con dominios de fonts |
| Health check fallaba en dashboard | Corregido endpoint a `/health` |
| `USER nginx` causaba permisos | Removido del Dockerfile |

### 4. Auditoria de Seguridad

Se confirmo que no hay secretos expuestos en:

- Codigo fuente del repositorio
- Configuracion de deployment
- Variables de entorno en Docker stack

### 5. Rediseno de la Pagina de Probabilidades

**Archivo:** `frontend/src/app/features/probabilities/probabilities.component.ts`

#### Dashboard de Estadisticas del Modelo

Se agrego un dashboard con 5 tarjetas informativas en la parte superior:

| Tarjeta | Descripcion |
|---------|-------------|
| Pares de emision | Cantidad total de pares (tag, word) con probabilidad calculada |
| Pares de transicion | Cantidad total de pares (prev_tag, next_tag) con probabilidad |
| Etiquetas unicas | Numero de etiquetas POS distintas en el modelo |
| Vocabulario | Tamano del vocabulario (palabras unicas) |
| Alpha de suavizado | Parametro de Laplace smoothing utilizado |

#### Tabla de Emision Mejorada

- Paginacion de 10 elementos por pagina con navegacion
- Descripciones de etiquetas EAGLES junto a cada codigo de etiqueta
- Badges de probabilidad por cada palabra asociada a una etiqueta
- Filtro de busqueda por etiqueta o palabra
- Exportacion a Excel (.xlsx)

#### Tabla de Transicion Mejorada

- Paginacion de 10 elementos por pagina con navegacion
- Barras de progreso visuales representando la magnitud de la probabilidad
- Filtro de busqueda por etiqueta previa o siguiente
- Exportacion a Excel (.xlsx)

#### Mapa de Calor Interactivo

El mapa de calor de la matriz de transicion recibio mejoras visuales e interactivas:

- **Fondo**: gradiente oscuro para mejor contraste
- **Gradiente de color**: 8 pasos de azul (del mas claro al mas intenso)
- **Crosshair**: guia visual de fila y columna al pasar el cursor
- **Hover**: efecto `scale(1.15)` para resaltar la celda activa
- **Click**: muestra detalle de la probabilidad seleccionada
- **Tamano de matriz**: 25x25 (top 25 etiquetas mas frecuentes)

#### Layout

- **Estructura**: 2 columnas (emision a la izquierda + transiciones a la derecha) con mapa de calor debajo
- **Ancho**: full-width sin restriccion `max-width`
- **Tamano**: compact sizing en toda la pagina
- **Eliminado**: layout anterior basado en pestanas (tabs)

### 6. Mejoras de UX en Entrenamiento

**Archivo:** `frontend/src/app/features/probabilities/probabilities.component.ts`

#### Banner de Entrenamiento Animado

Se reemplazo el badge inline sutil por un banner prominente y animado que aparece durante el entrenamiento del modelo:

- Efecto shimmer con animacion CSS `training-shimmer`
- Barra de progreso indeterminada con animacion `progress-indeterminate`
- Visualmente prominente para indicar claramente que el modelo se esta entrenando

#### Notificaciones Toast

Se agregaron notificaciones toast tras completar el entrenamiento:

- **Exito**: icono de check, mensaje de confirmacion, desaparece automaticamente
- **Error**: icono de error, mensaje descriptivo del fallo

## Como Desplegar

```bash
# Desarrollo local
cd frontend && npm install && ng serve
cd backend && pip install -r requirements.txt && uvicorn main:app --reload

# Produccion con Docker Swarm
docker stack deploy -c docker-stack.yml etiquetador

# Verificar servicios
docker service ls
docker service logs etiquetador_frontend
docker service logs etiquetador_backend
```

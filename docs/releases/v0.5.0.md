## Resumen

La version 0.5.0 representa la consolidacion del Etiquetador Morfosintactico HMM con mejoras significativas en calidad de codigo, confiabilidad y preparacion para despliegue. Se anadieron 30 unit tests, se dockerizo completamente el proyecto, se implemento suavizado de Laplace en el modelo HMM, se optimizo el algoritmo de Viterbi con un indice invertido O(1), y se corrigio una vulnerabilidad de path traversal.

### 2. Dockerizacion Completa

**`docker-compose.yml`:**
- Servicio `backend`: Python 3.12-slim, puerto 8000, volumenes para exports y cache
- Servicio `frontend`: multi-stage build (Node 20 + Nginx), puerto 4200
- Health check automatico via `urllib.request` al endpoint `/health`
- Dependencia ordenada: frontend espera a que backend este healthy

**`backend/Dockerfile`:**
```dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
RUN mkdir -p /app/cache /app/exports
EXPOSE 8000
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]
```

**`frontend/Dockerfile`:**
```dockerfile
FROM node:20-alpine AS build
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci
COPY . .

FROM nginx:alpine
COPY --from=build /app/dist/frontend/browser /usr/share/nginx/html

CMD ["nginx", "-g", "daemon off;"]
```

### 3. Suavizado de Laplace en HMM Trainer

**Archivo:** `backend/services/hmm_trainer.py`

El modelo HMM ahora usa suavizado de Laplace (add-alpha) para manejar palabras y transiciones no observadas en el corpus de entrenamiento:

**Emision:**
```
P(word|tag) = (C(tag, word) + alpha) / (C(tag) + alpha * V)
```
donde `V` es el tamano del vocabulario.

**Transicion:**
```
P(tag_i|tag_{i-1}) = (C(prev, next) + alpha) / (C(prev) + alpha * N)
```
donde `N` es el numero de etiquetas unicas.

- Parametro `smoothing` configurable en la funcion `train()` (default `1.0`)
- Usar `smoothing=0.0` para desactivar y obtener MLE clasico
- Garantiza que ninguna probabilidad sea exactamente 0

### 4. Optimizacion del Algoritmo de Viterbi

**Archivo:** `backend/services/viterbi_algorithm.py`

Se introdujo un indice invertido `word_to_tags` que mapea cada palabra a su conjunto de etiquetas posibles:

**Antes (O(|E|) por token):**
```python
# Recorria TODAS las emisiones para encontrar tags posibles
for (tag, word), prob in emission_probs.items():
    if word == token: ...
```

**Despues (O(1) por token):**
```python
# Lookup directo en indice pre-calculado
if word in word_to_tags:
    possible = set(word_to_tags[word])
```

El indice se construye durante el entrenamiento en `hmm_trainer.py` y se persiste en cache (`word_to_tags.pkl`). Para palabras desconocidas (OOV), se usan unicamente las etiquetas de clase abierta (N, V, A, R, Z, W).

### 5. Fix de Seguridad: Path Traversal

**Archivo:** `backend/routers/corpus.py`

El endpoint `POST /api/corpus/upload` aceptaba un parametro `corpus_dir` del cliente que podia ser usado para leer directorios arbitrarios del servidor. Se corrigio ignorando este parametro:

```python
background_tasks.add_task(
    _process_corpus,
    None,  # Siempre usa CORPUS_DIR del servidor
    request.max_files,
)
```

Adicionalmente, se removio un path absoluto de Windows hardcodeado en `config.py`, reemplazandolo por una variable de entorno con fallback seguro.

### 6. Accesibilidad (WCAG)

**Archivo:** `frontend/src/app/app.component.ts`

- **Skip link** (WCAG 2.4.1): enlace oculto "Saltar al contenido principal" que se hace visible al recibir foco con teclado, permitiendo saltar la navegacion
- **Landmark roles**: `role="main"` y `aria-label="Contenido principal"` en el area de contenido
- **Navegacion etiquetada**: `aria-label="Navegacion principal"` en la barra lateral

### 7. Migracion a Control Flow de Angular 19

**Archivos:** `navbar.component.ts`, `sidebar.component.ts`

Se migraron las directivas estructurales clasicas al nuevo control flow nativo de Angular 19:

| Antes | Despues |
|-------|---------|
| `*ngIf="isDark"` | `@if (isDark) { ... }` |
| `*ngFor="let item of items"` | `@for (item of items; track item.route) { ... }` |

Esto mejora el rendimiento (no requiere importar `CommonModule` para el control flow) y se alinea con las mejores practicas de Angular 19.
